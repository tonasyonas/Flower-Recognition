\subsection{Problem Motivation}

Fine-grained image classification involves distinguishing between visually similar categories such as bird species, dog breeds, or flower types. Unlike generic object recognition, these tasks require models to capture subtle discriminative features while remaining robust to variations in lighting, orientation, and occlusion.

The Oxford Flowers 102 dataset \citep{nilsback2008} exemplifies this challenge: with only 10 images per class in the standard split, models have very few training samples to learn parameters that accurately classify 102 flower categories. The constraint makes Flowers 102 a natural benchmark for evaluating few-shot learning methods, where models must generalise effectively from limited data.

While modern architectures such as Convolutional Neural Networks (CNNs) and Vision Transformers (ViTs) have achieved strong performance on large-scale datasets, it is unclear whether those gains transfer to low-data conditions. Their effectiveness under data-scarce regimes, where only $k$ examples per class are available ($k \in \{1, 2, 5, 10\}$), remains less well understood.

\subsection{Approach Overview}

This project investigates the role of architectural design and parameter-efficient adaptation in few-shot learning. Under controlled experimental conditions, we compare:

\begin{itemize}
  \item \textbf{Five ResNet18-based architectures}: a baseline plus four advanced convolutional variants (dilated, deformable, depthwise-separable, and a novel hybrid combining dilated and deformable layers).
  \item \textbf{Four regularization strategies} on the ResNet18 baseline: freezing early layers, label smoothing, MixUp, and stronger augmentation.
  \item \textbf{A larger CNN backbone} (ResNet50) to test whether scaling parameters helps.
  \item \textbf{Visual Prompt Tuning (VPT)} on a ViT-B/16 backbone, in both Shallow and Deep configurations \citep{jia2022vpt}.
\end{itemize}

All models share an identical training recipe (AdamW, cosine annealing, 50 epochs, batch size 32) so that differences are attributable to the method itself. Performance is measured by accuracy, mean per-class accuracy, and top-5 accuracy on the test set, with a few-shot sweep at $k = 1, 2, 5$ to study how each method degrades.

Through this study, we ask whether architectural complexity translates into better generalisation in the few-shot regime, or whether simpler approaches with strong pre-training are more effective. We also test whether parameter-efficient tuning can match or beat full fine-tuning on a fine-grained task.

\subsection{Training and Evaluation Setup}

We use the canonical \texttt{torchvision.datasets.Flowers102} split: 1020 training images, 1020 validation images, and 6149 test images. Images are resized to $224 \times 224$ and normalised with ImageNet statistics. The training transform applies a random horizontal flip and rotation up to 15 degrees.

We report three test-set metrics:
\begin{itemize}
  \item \textbf{Accuracy}: overall correct predictions over all samples.
  \item \textbf{Mean per-class accuracy (MPC)}: average per-class recall, our primary metric following \citet{nilsback2008}. The test set is class-imbalanced (20 to 258 images per class), so this corrects for class frequency.
  \item \textbf{Top-5 accuracy}: fraction of samples where the true class appears in the top-5 predictions.
\end{itemize}

Unless stated otherwise, every model trains for 50 epochs with AdamW ($\text{lr} = 10^{-3}$), cosine annealing, batch size 32, automatic mixed precision, and seed 69. Experiments run on a single NVIDIA RTX 3070 Ti.
