\subsection{Dataset and Preprocessing}

We use the canonical Oxford Flowers 102 split \citep{nilsback2008}: 1020 training images (10 per class), 1020 validation images (10 per class), and 6149 test images (20--258 per class). All images are resized to $224 \times 224$ and normalised with ImageNet statistics. The training transform applies a random horizontal flip and rotation up to $\pm 15^\circ$.

\subsection{CNN Architectures}

All seven CNN configurations start from a ResNet pre-trained on ImageNet-1k. We replace the original classifier with a 102-class linear head. Wherever we modify a layer, we copy the pre-trained weights into the new layer when the kernel shape allows. Table~\ref{tab:cnn-arch} summarises the seven configurations.

\subsection{Visual Prompt Tuning}

We implement VPT on a frozen ViT-B/16 (86M backbone parameters) using \texttt{torchvision}'s pre-trained weights. VPT-Shallow inserts 10 learnable prompt tokens at the first Transformer layer only, so the input becomes $[x_0, P, E_0]$ where $P \in \mathbb{R}^{10 \times 768}$. Subsequent layers see $[x_i, Z_i, E_i]$ unchanged. Only $P$ and the 102-way head are trained (86,118 parameters total). VPT-Deep inserts a fresh set of 10 prompts at the input of every Transformer layer, with prompts from the previous layer discarded at each step. Trainable parameter count is 170,598, or 0.2\% of the backbone. Both variants use truncated-normal initialisation ($\sigma = 0.02$) for the prompts.

\subsection{Training and Augmentation}

Unless stated otherwise, every model trains for 50 epochs (30 for few-shot) with AdamW ($\text{lr} = 10^{-3}$), cosine annealing ($T_{\max}$ = epochs), batch size 32, mixed precision, Cross-Entropy loss, and seed 69. Standard augmentation is the resize, flip, and rotation pipeline described above. The optional strong pipeline replaces it with random resized cropping (scale 0.6 to 1.0) and colour jitter (brightness, contrast, and saturation 0.2; hue 0.1). MixUp ($\alpha = 0.2$) is applied through the official \texttt{torchvision v2.MixUp} integrated into a multi-worker-safe collate function.

\subsection{Few-Shot Protocol}

For $k \in \{1, 2, 5\}$, we deterministically subsample the training set to the first $k$ images per class (seed 69) and train for 30 epochs with the same optimiser and schedule. Validation and test sets remain unchanged. The full $k = 10$ setting uses the canonical training split and 50 epochs.
